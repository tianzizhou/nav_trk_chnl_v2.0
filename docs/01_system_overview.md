## 2 引言

### 2.1 编写目的

本文档是多模态融合反无人机探测分系统的总体技术方案，旨在完整描述系统的
设计思路、架构方案、关键技术路线及性能指标，为后续详细设计、原型开发和
系统集成提供顶层指导。

### 2.2 项目背景

近年来，低慢小（Low-Slow-Small, LSS）无人机对要地防护构成日益严峻的
威胁。传统反无人机探测手段（雷达、光电）在实际作战中面临众多能力短板，
经系统性研究梳理，**反无探测分系统存在以下十二类典型问题**：

#### 第一类：超低空飞行（h < 10 m）

雷达地面反射形成强杂波（杂噪比 CNR 可达 20~40 dB），目标回波被
完全淹没。MTI 对消后仍有残余杂波，OS-CFAR 门限被杂波抬高，
目标检测概率从正常的 80% 以上骤降至 5% 以下。光电虽不受杂波
影响，但超低空目标与地面背景混为一体，可见光检测器的信背比（SBR）
显著下降，红外通道受地面热辐射干扰，检测可靠性下降。

#### 第二类：近距快速飞越（R < 200 m）

目标距离近时角速度极大。以 50 m/s 速度横切200m距离为例，
目标角速度可达 14.3°/s，当目标飞到 100m 时角速度达 28.6°/s。
光电窄视场（FOV ≈ 2.7°）在此角速度下，目标在 0.1~0.2s 内
即飞出视场。云台伺服系统无法实时跟上，导致目标脱离跟踪。
同时，近距雷达虽可检测到目标，但仅提供测距信息，角度引导
精度不足以维持光电锁定。

#### 第三类：蛙跳机动（飞行-悬停-飞行）

悬停阶段目标径向速度为零，雷达基于 MTI/MTD 的检测体制将悬停
目标与地杂波一同对消，导致检测失效。传统雷达仅能在目标飞行
阶段建立航迹，悬停阶段航迹丢失，目标重新起飞后无法与之前
航迹关联，造成反复的"新目标告警"和航迹碎片化。

#### 第四类：城市复杂背景

楼宇密集区域存在严重的电磁多径效应和光学遮挡。雷达信号被
建筑物反射形成虚假目标（ghost target），杂波环境复杂且非
平稳；光电视线被建筑物遮挡，目标在楼宇间穿梭时频繁丢失。
两种传统传感器均难以维持连续跟踪。

#### 第五类：夜间/低照度环境

夜间可见光相机性能严重下降，对比度降低导致小目标检测率
不足 30%。传统光电系统仅配备可见光相机时，夜间基本失去
目标发现能力，仅能依赖雷达独立工作，系统冗余度降为零。

#### 第六类：无人机集群/蜂群攻击

多架无人机同时来袭时（典型 5~20 架），单部光电云台只能
跟踪 1 个目标，雷达虽可同时检测多目标但数据关联复杂度
急剧上升（JPDA 计算量呈指数增长），系统面临"发现得了、
跟不过来"的困境。

#### 第七类：隐身/低 RCS 无人机

新型复合材料无人机 RCS 可低至 0.001 m² 以下，雷达在
3000m 处的 SNR 不足以检测。加之近年出现的泡沫/3D 打印
机体对雷达波吸收性强，传统雷达探测距离显著缩短。

#### 第八类：电子对抗（ECM）与 GPS 欺骗

敌方可对雷达实施有源干扰（噪声压制、欺骗干扰），降低
雷达检测性能。同时，可对本方系统 GPS 授时进行欺骗或干扰，
影响时间同步精度和目标定位准确性。

#### 第九类：地形遮蔽（山谷、丘陵、树林）

在非城市但地形复杂的区域（如山谷、丘陵、密林边缘），
雷达和光电均存在视线被地形遮挡的问题。目标可利用地形
起伏进行规避机动，使传统传感器间歇性丢失目标。

#### 第十类：高速俯冲突防

部分攻击型无人机在末段采用大角度高速俯冲（速度可短时
超过 50 m/s 达到 80~100 m/s），从高空快速接近。此时
光电跟踪角速率需求极大，雷达径向速度超出不模糊范围，
系统从发现到引导打击的时间窗口极短（仅数秒）。

#### 第十一类：与合法飞行器/鸟类共存

探测区域内可能存在合法航拍无人机、飞鸟、气球、风筝等
非威胁目标，系统需具备有效区分能力，否则将产生大量
虚警，严重影响作战效率和操作员信任度。

#### 第十二类：恶劣电磁环境

在部队驻地、城市或工业区附近，2.4 GHz 和 5.8 GHz 频段
存在大量 WiFi、蓝牙等民用信号，RF 侦测面临高密度同频
干扰，虚假信号率高，对无人机遥控信号的检测造成困难。

---

**小结**：上述十二类问题表明，任何单一传感器均无法在所有场景下
可靠工作。本方案提出 **"雷达 + 光电 + 声学阵列 + 无线电侦测"**
四模态传感器融合探测方案，通过多源信息互补与数据融合，系统性
解决上述难题。

## 3 系统需求

### 3.1 作战需求

| 序号 | 需求项 | 指标要求 | 备注 |
|------|--------|---------|------|
| 1 | 目标类型 | 低慢小飞行器（多旋翼/固定翼无人机） | RCS: 0.001 ~ 0.01 m² |
| 2 | 目标速度 | ≤ 50 m/s | 含悬停（0 m/s） |
| 3 | 飞行高度 | 10 ~ 300 m | 含超低空场景 |
| 4 | 探测距离 | ≥ 3000 m（最远） | 雷达主导 |
| 5 | 方位覆盖 | 360° | 全向警戒 |
| 6 | 定位精度 | CEP ≤ 5 m @ 1000 m | 满足打击引导需求 |
| 7 | 反应时间 | 发现→确认 ≤ 3 s | 首次报告延迟 |
| 8 | 航迹更新率 | ≥ 5 Hz | 满足火控解算需求 |
| 9 | 虚警率 | ≤ 1 次/小时 | 减少误报干扰 |
| 10 | 多目标能力 | 同时跟踪 ≥ 10 批目标 | - |

### 3.2 平台约束

| 序号 | 约束项 | 要求 |
|------|--------|------|
| 1 | 安装平台 | 东风猛士车载 |
| 2 | 系统成本 | ≤ 100 万元/套 |
| 3 | 总功耗 | ≤ 2000 W |
| 4 | 总重量 | ≤ 500 kg（含结构件） |
| 5 | 工作温度 | -30°C ~ +55°C |
| 6 | 部署方式 | 无需展开，**开机即进入工作状态** |
| 7 | 行进间工作 | 车辆行驶中具备探测定位能力 |

**关于部署方式的说明**：

系统采用固装一体化设计，所有传感器和处理平台永久安装于车顶
集成桅杆和车内机柜中，**不存在展开/撤收过程**。车辆启动后
系统上电自检（< 30 s），自检通过即自动进入全向警戒模式。

**关于行进间工作的技术要求**：

系统需在车辆行进状态下（速度 ≤ 80 km/h）保持探测定位能力，
为此需满足以下技术条件：

1. **惯性导航/GPS 组合定位**：车载 INS/GNSS 组合导航系统
   实时提供车辆位置、航向和姿态，精度优于 0.1° 航向、
   0.5 m 位置（RTK 模式）。
2. **光电云台稳定**：云台配备三轴陀螺稳定系统，车辆行驶
   引起的振动和姿态变化由稳定平台补偿，视线稳定精度
   优于 0.1 mrad (RMS)。
3. **声学修正**：车辆行进时自身噪声（发动机、路噪）约
   70~85 dB，需通过自适应噪声对消和参考麦克风方法抑制
   本车噪声，行进间声学探测距离降至约 100~150 m。
4. **坐标系实时修正**：所有传感器量测需实时补偿车辆位移，
   将量测从车体坐标系转换到地理坐标系，补偿公式：
   \(\mathbf{p}_{geo} = \mathbf{R}_{body}^{geo}(t) \cdot \mathbf{p}_{body} + \mathbf{p}_{vehicle}(t)\)
5. **雷达杂波补偿**：车辆移动导致雷达杂波频谱展宽，需根据
   车辆速度进行地杂波补偿（GMC, Ground Moving Clutter
   Compensation），修正多普勒偏移：
   \(\Delta f_{GMC} = \frac{2 v_{vehicle} \cos\theta}{\lambda}\)

### 3.3 环境适应性需求

系统需在以下环境条件下保持探测能力：
- 风速 ≤ 20 m/s（声学受限时自动降权）
- 能见度 ≥ 100 m（光电受限时自动降权）
- 降雨强度 ≤ 16 mm/h（中雨）
- 城市、郊野、山地等多种地形

### 4.3 工作模式

系统支持以下工作模式：

| 模式 | 说明 | 传感器状态 |
|------|------|-----------|
| **全向警戒** | 360° 搜索，发现新目标 | 全部传感器工作 |
| **定向搜索** | 指定扇区重点搜索 | 雷达定向 + 光电引导 |
| **跟踪锁定** | 对已确认目标持续跟踪 | 光电锁定 + 多源更新 |
| **打击引导** | 为打击系统提供实时目标参数 | 高更新率模式（50Hz）|
| **静默模式** | 仅被动传感器工作 | 关闭雷达，声学+RF+光电 |

### 5.3 传感器选型论证与角色分析

本系统选择四种传感器并非简单堆砌，而是经过系统性论证，每种传感器
有其明确的加入目的、所解决的问题和在系统中的角色定位。

#### 5.3.1 FMCW 雷达 — 为什么需要？

**加入目的**：提供远距离、全天候、具备测距测速能力的主动探测手段，
是系统的"千里眼"和"距离尺"。

**解决的问题**：
- 第七类（低RCS目标）：相控阵天线增益集中，FMCW 体制相参积累增益
  高，对小 RCS 目标探测距离可达 3000 m
- 第五类（夜间）：雷达不受光照条件影响，7×24 全天候工作
- 第六类（集群攻击）：雷达可在一个扫描周期内同时检测多个目标，
  无需逐一引导
- 第三类（蛙跳，飞行阶段）：飞行阶段雷达可有效检测

**系统中仍存在的残余问题**：
- 超低空（第一类）：强地杂波导致检测概率骤降
- 蛙跳悬停阶段（第三类）：零速目标被 MTI 对消（微多普勒可部分缓解）
- 城市多径（第四类）：虚假目标干扰
- 近距无角度盲区但精度有限（3°方位精度在近距下偏低）
- 被 ECM 干扰风险（第八类）

**角色定位**：**远距主传感器 + 测距基准**
- 距离 > 500 m 时为主要探测手段
- 提供唯一的斜距和径向速度量测
- 为光电提供初始引导方向

**使用条件与搭配**：

| 场景 | 雷达角色 | 搭配方式 |
|------|---------|---------|
| 开阔/中远距 | 主探测 | 雷达发现 → 引导光电确认 |
| 超低空 | 辅助（降权） | 声学/RF 主导，雷达杂波图模式辅助 |
| 近距 (<200m) | 测距 | 雷达提供距离，广角+声学提供角度 |
| 蛙跳-飞行阶段 | 主探测 | 雷达跟踪 + 光电锁定 |
| 蛙跳-悬停阶段 | 辅助 | 微多普勒检测 + 声学主导 |
| 城市 | 有条件使用 | 仅在非遮挡区域有效，声学/RF补充 |
| 夜间 | 主探测 | 雷达+红外+声学组合 |

#### 5.3.2 光电系统 — 为什么需要？

**加入目的**：提供高精度角度量测和目标视觉识别/分类能力，是系统的
"鹰眼"和"识别器"。

**解决的问题**：
- 角度精度瓶颈：雷达方位精度仅 3°，1000m 处横向误差 ~52m，
  无法满足打击引导需求。光电角度精度 0.5 mrad，1000m 处横向
  误差仅 0.5m，**精度提升 100 倍**
- 第十一类（目标分类）：YOLOv8 深度学习可区分无人机/鸟类/其他，
  是分类融合中最可靠的信息源
- 目标确认：人工通过光电画面进行目标最终确认，是开火决策的
  关键依据

**系统中仍存在的残余问题**：
- 近距脱跟（第二类）：窄视场 FOV 不足（通过增加广角相机缓解）
- 雨雾天气（能见度<1km时性能严重下降）
- 夜间可见光失效（通过增加红外通道缓解）
- 超低空背景融合（第一类）：目标与地面难区分
- 不提供距离信息：仅有角度量测，单独无法定位
- 只能跟踪 1 个目标（云台物理限制）

**角色定位**：**高精度角度传感器 + 目标识别器 + 人工确认终端**
- 提供打击引导所需的高精度角度
- 目标分类最高置信度来源
- 人机交互的视觉窗口

**使用条件与搭配**：

| 场景 | 光电角色 | 搭配方式 |
|------|---------|---------|
| 开阔/中远距 | 辅助跟踪 | 雷达引导指向 → 光电精跟踪 |
| 超低空 | 辅助（受限） | 声学引导 → 光电捕获确认 |
| 近距 (<200m) | 广角接替 | 窄视场切广角，声学辅助角度 |
| 打击引导 | 核心 | 高精度角度 → 火控解算 |
| 夜间 | 红外主导 | 红外通道替代可见光 |
| 城市 | 有条件使用 | 非遮挡区域内有效 |

#### 5.3.3 声学阵列 — 为什么需要？

**加入目的**：填补雷达和光电在超低空、悬停、近距、城市遮挡等
场景下的探测盲区，是系统的"顺风耳"。

**解决的问题**：
- 第一类（超低空）：声波不受地杂波影响，超低空检测可靠
- 第三类（蛙跳悬停）：旋翼噪声在悬停时持续存在，是检测悬停
  目标的最可靠手段
- 第二类（近距脱跟）：声学阵列全向接收、无视场限制，近距
  角速度大时仍可连续提供 DOA
- 第四类（城市遮挡）：声波可绕射建筑物（波长 ~0.04~3.4m，
  远大于光波），在部分遮挡条件下仍可检测目标
- 第十一类（分类辅助）：无人机旋翼噪声频谱特征明显，可有效
  区分无人机和鸟类

**系统中仍存在的残余问题**：
- 探测距离有限（安静环境 ~300m，城市 ~150m）
- 不提供距离信息（仅方向角）
- 环境噪声影响大（风噪、城市噪声、车辆行进噪声）
- 精度有限（方位 3°、俯仰 8°）
- 多个声源时分辨能力受限

**角色定位**：**近距/困难场景补充探测器 + 悬停目标专用探测器**
- 300m 内的可靠近程警戒
- 雷达失效场景的关键替补
- 悬停目标的"最后一道网"

**使用条件与搭配**：

| 场景 | 声学角色 | 搭配方式 |
|------|---------|---------|
| 超低空 | **主探测** | 声学发现 → 引导光电确认 |
| 蛙跳-悬停 | **主探测** | 声学连续跟踪 + 光电保持 |
| 近距 (<200m) | **主角度** | 声学DOA + 雷达测距 + 广角验证 |
| 城市遮挡 | **主探测** | 声学穿透 + RF辅助，出遮挡后雷达接力 |
| 开阔/远距 | 无效 | 超出探测距离，不参与 |
| 行进中 | 受限 | 本车噪声降低有效距离至100~150m |

#### 5.3.4 无线电（RF）侦测 — 为什么需要？

**加入目的**：利用无人机遥控/图传链路信号实现被动探测和目标识别，
是系统的"电子哨兵"。

**解决的问题**：
- 第一类（超低空）：RF 信号不受地形和杂波影响
- 第四类（城市遮挡）：电磁波可在建筑物间传播（反射/绕射），
  遮挡影响远小于光电
- 第十一类（目标识别）：通过协议识别可直接判定无人机品牌/型号
  （DJI、ELRS 等），分类置信度极高
- 预警前置：RF 信号传播距离通常远于声学，可在 1500m 处提供
  早期预警
- 遥控器定位：可同时检测无人机和地面操控人员方向

**系统中仍存在的残余问题**：
- 自主飞行无人机完全无效（第七类）：无遥控信号时 RF 侦测
  检测概率为零
- 仅提供方位角（1维量测），无俯仰和距离
- 2.4/5.8 GHz 频段民用信号密集（第十二类），虚警率高
- 跳频/扩频信号截获困难
- 测向精度有限（~5°）

**角色定位**：**被动早期预警 + 协议识别器 + 声学的中距补充**
- 提供 500~1500m 的被动探测能力
- 弥补声学 300m 以远的探测空白
- 协议识别为分类融合提供高置信信息

**使用条件与搭配**：

| 场景 | RF 角色 | 搭配方式 |
|------|--------|---------|
| 开阔/远距 | 辅助预警 | RF 预警 → 引导雷达定向搜索 |
| 超低空 | 辅助 | RF+声学联合确认 → 光电引导 |
| 城市遮挡 | 辅助 | RF 穿透 + 声学绕射联合覆盖 |
| 自主飞行 | **无效** | 权重置零，其他三传感器补偿 |
| 电磁密集区 | 受限 | 提高检测门限，降低虚警 |

- 声学阵列通道数可从 32 降至 16，节省 ~3 万，方位精度从 3° 降至 5°
- RF 侦测模组可选低端方案，节省 ~3~5 万
- 若不需要自主飞行无人机检测能力，可去除 RF 侦测模组，节省 10~15 万
- 融合处理平台可用国产替代方案（如华为昇腾），节省 ~2 万

## 8 与外部系统接口

### 8.1 接口总览

```
                          ┌──────────────┐
                          │  指控系统    │
                          │  (上级)      │
                          └──────┬───────┘
                                 │ 数据链/以太网
                          ┌──────▼───────┐
                          │  探测分系统  │ ←── 本系统
                          └──────┬───────┘
                                 │
                 ┌───────────────┼───────────────┐
                 │               │               │
          ┌──────▼───┐   ┌──────▼───┐   ┌──────▼───┐
          │ 火工打击 │   │ 激光打击 │   │ 其他打击 │
          │ 分系统   │   │ 分系统   │   │ 分系统   │
          └──────────┘   └──────────┘   └──────────┘
```

### 8.2 接口类型

| 接口 | 物理层 | 协议 | 数据内容 |
|------|--------|------|---------|
| 指控上行 | 千兆以太网 | UDP/TCP | 指挥控制指令 |
| 指控下行 | 千兆以太网 | UDP | 航迹报告、态势信息 |
| 打击引导 | 千兆以太网/RS422 | 自定义协议 | 目标参数（位置/速度/精度） |
| GPS/北斗 | 串口 RS232 | NMEA/RTCM | 授时 + 定位 |
| 内部总线 | 千兆以太网 | UDP | 传感器量测数据 |
